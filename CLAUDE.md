# あの人の裏の顔診断（ちょいゲス）

## 開発方針
- Vite + React (JavaScript) で構成
- 依存: `react-router-dom`, `tailwindcss`, `@tailwindcss/vite`
- CSS は Tailwind CSS v4 ユーティリティ + index.css でカスタムテーマ定義
- データは JSON で外部化し、テキスト差し替えが容易な設計
- カラー: メイン #0D0808（ダークブラック）、アクセント #CC1133（ダーククリムゾン）
- 判定ロジックは内部的に8軸スコアリングを使用（UIには一切表示しない）

## コマンド
```bash
npm install        # 依存インストール
npm run dev        # 開発サーバー起動 (localhost:5173)
npm run build      # プロダクションビルド
npm run preview    # ビルド結果プレビュー
```

## 判定ロジック（src/utils/scoring.js）
1. 回答値(1-5)を中央3基準で -2〜+2 に正規化（buttonToScore）
2. 各質問のaxesオブジェクトに基づき、weight × answerScore を全8軸に加算
3. 性格分析4軸: 各軸スコアが 0以上 → 正方向の文字、0未満 → 負方向の文字
4. 4文字を結合して内部typeKeyを生成 → types.json から結果取得
5. URLでは typeKeyToId() で数値ID（0-15）に変換し、内部キーを隠蔽
6. Spice 4軸: 最大スコアの軸と値からModifier（修飾語）を決定
7. 出力: `{{Modifier}}{{Noun}}`（例: 「加工上手の路地裏チワワ」）
8. 修飾語にはそれぞれ詳細解説（MODIFIER_DETAILS）があり、結果画面に統合表示
9. フォールバック: どの回答組み合わせでも必ず結果が生成される

## 結果画面の構成（あの人の裏の顔）
1. 修飾語 + 動物タイプ名 & SVGイラスト（修飾語とタイプ名は同一スタイルで表示）
2. 表の顔（front）
3. 裏の顔（hidden）+ 修飾語の詳細解説
4. この人のトリセツ（manual）— 攻略法と地雷ポイント
5. デートのシミュレーション（date）— おすすめスポット付き
6. 相手の心を開かせるキラーフレーズ（killer）
7. ハッシュタグ: #あの人の裏の顔診断

## UIデザイン
- フルスクロール1カラム（画面遷移なし）
- 丸型ぷるんボタン（角ばったボタンなし）
- ビビッドピンク (#FF3355) でゲスい強調
- SNSシェア対応（X/コピー）
- 質問シャッフル: アクセスごとにランダム順、同じ軸が連続しない
- 未回答バリデーション: 送信時に未回答項目を緑色ハイライト + 自動スクロール
- URL形式: `/result?t={数値ID}&m={修飾語}`（内部キーを隠蔽）
